# tinyfs Implementation Plan

Spec: ../specs/2026-07-18-tinyfs-design.md
Status: complete

## Milestones

- [x] **M1 — crate skeleton + on-disk types.** Workspace members `tinyfs`
  and `tools/mkfs-tinyfs`; `layout.rs` ser/de with CRC-32, `BlockDevice`
  trait + `MemDevice`, `path.rs`. Verified: `cargo test -p tinyfs`
  round-trips; `cargo build -p tinyfs --target aarch64-unknown-uefi`
  proves no_std cleanliness.
- [x] **M2 — full FS logic on MemDevice.** format/mount, read/list, then
  write/mkdir/rm/mv, in-RAM bitmap + pending_free, commit. Crash test cuts
  the write log at every prefix: each cut mounts as old or new tree, never
  corrupt; torn-checkpoint test falls back a generation.
- [x] **M3 — mkfs-tinyfs.** create/--populate/ls/cat/check over a file
  device; Makefile `disk.img` target (create-if-missing, `cleandisk`).
- [x] **M4 — virtio-blk + QEMU wiring.** `virtio.rs` split into
  `Transport`/`Queue` + input `VirtioDevice`; new `virtio_blk.rs`
  (modern 0x1042, FLUSH negotiated, INTx masked); `drivers::probe()` single
  scan/allocator. Verified on aarch64/HVF and x86_64/tcg: blk capacity
  logged, input still works.
- [x] **M5 — mount + shell.** `kernel/src/fs/` glue, mount-by-magic at boot,
  terminal cwd + `ls cat write append mkdir rm mv cd pwd fsinfo`. Verified
  end-to-end via QEMU monitor sendkey: wrote files in-guest, `cat` after
  reboot, cross-arch (image written on aarch64 mounts on x86_64), host
  `mkfs-tinyfs check` clean.
- [x] **M6 — docs.** Spec + this plan + README note.

## Gotchas discovered

- A polled virtio-blk device left INTx asserted → interrupt storm on the
  shared level-triggered PCI line the moment `irq::init()` ran (aarch64
  hang after "starting scheduler"). Fix: PCI command bit 10 (INTx disable)
  for polled devices.
- QEMU `virtio-blk-pci` defaults to transitional id 0x1001;
  `disable-legacy=on` is required to enumerate as modern 0x1042 (which also
  usefully distinguishes our disk from the VVFAT ESP on aarch64).
- Never `cargo build --target <uefi>` at the workspace root now that a
  host-target std binary is a member; the Makefile builds with `-p`.
