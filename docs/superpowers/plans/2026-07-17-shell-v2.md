# Shell v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the macOS-homage shell with a modern soft-dark-neutral windowed desktop (multi-window, snapping, resizing, command palette) per `docs/superpowers/specs/2026-07-17-shell-v2-design.md`.

**Architecture:** New `ui/shell/` module: a window manager (`wm.rs`) hosting `App` trait objects, plus dock, status bar, palette, and mesh-gradient wallpaper. App logic (notes/monitor/clock/calc) is recovered from the discarded Deck branch — commit `fb45af2`, reachable via reflog — and adapted from the `Card` trait to the `App` trait. Old `ui/desktop.rs`/`ui/wallpaper.rs` are deleted in the final task.

**Tech Stack:** existing tinyOS kernel (Rust nightly no_std, aarch64/x86_64 UEFI), fontdue, QEMU/QMP verification harness.

## Global Constraints

- Tokens (spec, verbatim): field `#0F1222`, blob-a `#7C5CFF`, blob-b `#2BB8D9`, surface `#1C1E26`, surface-hi `#252834`, border white@8%, accent `#7C5CFF`, text `#E6E8F0`, text-dim `#8A8FA3`.
- Radii: 14px windows/panels, 10px icon tiles. No traffic lights: one ghost `×` on the RIGHT of the inline title row. Focused window = 1.5px accent border (draw 2px), unfocused = white@8%.
- Keyboard: `Ctrl+K` palette, Esc dismiss, `Ctrl+Left/Right` snap halves, `Ctrl+Up` maximize, `Ctrl+Down` restore. Ctrl keycode 29 (`keys::LCTRL` exists).
- Recovery source: `git show fb45af2:kernel/src/<path>` for apps/calc/palette logic. Adapt, don't rewrite.
- Test cycle: `make build 2>&1 | grep -E '^error' -A6` silent, then the repo's standard headless QMP boot (aarch64, HVF, ramfb, 1440x900, `-qmp unix:/tmp/tinyos-qmp.sock,server,nowait`), driving `input-send-event` and `screendump`, then Read the PNG. `make firmware` once per session.
- Commit after every task.

- Glyphs: terminal `>_`, notes `≡` (use "=" if glyph missing), monitor `~`, clock `◷` (fallback "()"): use ASCII-safe: ">_", "N", "M", "C" — pick mono-renderable ASCII.
- `main.rs`: same loop as today but `ui::shell::Shell` instead of `ui::desktop::Desktop`; keep `FB_SIZE`, input poll, present, pacing. Old desktop mod stays compiling until Task 4.

### Task 2 (W2): Multi-window manager + recovered apps

**Files:**
- Create: `kernel/src/ui/shell/wm.rs`, `kernel/src/apps/notes.rs`, `kernel/src/apps/monitor.rs`, `kernel/src/apps/clock.rs`
- Modify: `kernel/src/ui/shell/mod.rs`, `kernel/src/apps/mod.rs`, `kernel/src/main.rs` (stats feed)

**Interfaces:**
- Recover app logic: `git show fb45af2:kernel/src/apps/notes.rs` (drop Card impl → App impl; keep line-editing engine verbatim), same for `monitor.rs` (RingBuf + tick + sparkline; label() calls become plain `fonts.ui_medium.draw` 13px TEXT_DIM) and `clock.rs` (timer state machine; SIGNAL→ACCENT).
- `wm.rs` owns `pub struct Wm { pub windows: Vec<Window>, pub focus: usize, drag: Option<DragState>, ... }`:
  - `open(&mut self, app: Box<dyn App>, screen: (i32,i32))` — cascade placement (64 + 32*n offsets), size = preferred clamped to screen minus bars; pushes + focuses (last = topmost).
  - `focus_or_open(name, factory)` single-instance rule for terminal/monitor/clock; notes always opens new.
  - `close(idx)`, `bring_to_front(idx)` (Vec rotate: remove+push, focus = last).
  - `hit_test(px,py) -> Option<(idx, Zone)>` topmost-first; `Zone::{Title, Close, Body, ResizeGrip}` (grip = 18px bottom-right square; close = 22px circle at title right).
  - `on_pointer(...)`/`on_click`/`on_release` handling drag-move (Title) with per-window drag offset.
  - Preferred sizes: notes (420, 320), monitor (420, 360), clock (300, 200).
- `Shell` routes: click → wm hit test (close/front/drag) else dock hit else nothing; keys → focused app (keycode_to_char w/ shift); per-frame `stats_tick(events)` forwarding to any open `MonitorCard`-equivalent (`MonitorApp`) via `as_any().downcast_mut`.
- `timer` command arrives with palette (Task 4); until then clock is clock-only.
- Draw order: wallpaper → windows in vec order (focused = last = drawn on top) → status bar → dock → cursor.

### Task 3 (W3): Resize grip + snapping

**Files:**
- Modify: `kernel/src/ui/shell/wm.rs`, `kernel/src/ui/shell/mod.rs`

**Interfaces:**
- DragState grows: `enum DragKind { Move, Resize }`; Resize adjusts rect.w/h from pointer minus grip anchor, clamped to `app.min_size()` and screen.
- Snap fields on `Window`: `restore: Option<Rect>` (pre-snap geometry).
- During Move drag: if pointer.x < 16 → preview left half; > width-16 → right half; pointer.y < STATUS_H+8 → maximize preview. Preview = 2px ACCENT outline rounded rect over the zone (drawn in compose while dragging). On release in zone: save `restore = Some(old rect)`, set rect to zone (halves exclude status bar top and dock band bottom 90px; maximize = same vertical bounds, full width minus 16px margins).
- Starting a Move drag on a snapped window (restore.is_some()): restore size immediately under the pointer (rect = restore centered on pointer x, title under cursor), clear restore.
- Keyboard: Ctrl+Left/Right/Up applies same zones to focused window; Ctrl+Down restores if `restore` set.
- `keys::UP/DOWN/LEFT/RIGHT` already exist.
